Good moment to introduce **`HttpResponse` + `ResponseBuilder`** properly.
The goal is to separate:

```
ResponseBuilder → decides WHAT to send
HttpResponse    → knows HOW to format HTTP
ConnectionHandler → does network I/O
```

Your hardcoded response becomes a structured object that later supports:

* status codes
* headers
* body
* keep-alive
* CGI
* static files

---

# 1️⃣ HttpResponse class

Example minimal implementation compatible with **C++98**.

### Header

```cpp
#ifndef HTTPRESPONSE_HPP
#define HTTPRESPONSE_HPP

#include <string>
#include <map>

class HttpResponse
{
public:
    HttpResponse();

    void setStatus(int code, const std::string& message);
    void setHeader(const std::string& key, const std::string& value);
    void setBody(const std::string& body);

    std::string serialize() const;

private:
    int status_code_;
    std::string status_message_;
    std::map<std::string, std::string> headers_;
    std::string body_;
};

#endif
```

---

# 2️⃣ HttpResponse implementation

```cpp
#include "HttpResponse.hpp"
#include <sstream>

HttpResponse::HttpResponse()
    : status_code_(200), status_message_("OK")
{
}

void HttpResponse::setStatus(int code, const std::string& message)
{
    status_code_ = code;
    status_message_ = message;
}

void HttpResponse::setHeader(const std::string& key, const std::string& value)
{
    headers_[key] = value;
}

void HttpResponse::setBody(const std::string& body)
{
    body_ = body;
}

std::string HttpResponse::serialize() const
{
    std::ostringstream response;

    // Status line
    response << "HTTP/1.1 "
             << status_code_
             << " "
             << status_message_
             << "\r\n";

    // Headers
    for (std::map<std::string,std::string>::const_iterator it = headers_.begin();
         it != headers_.end();
         ++it)
    {
        response << it->first << ": " << it->second << "\r\n";
    }

    // End headers
    response << "\r\n";

    // Body
    response << body_;

    return response.str();
}
```

# 3️⃣ ResponseBuilder class

This class **decides the response logic**.

### Header

```cpp
#ifndef RESPONSEBUILDER_HPP
#define RESPONSEBUILDER_HPP

#include "HttpResponse.hpp"
#include "HttpRequest.hpp"

class ResponseBuilder
{
public:
    HttpResponse build(const HttpRequest& request);
};

#endif
```

---

# 4️⃣ ResponseBuilder implementation

Simple example responding like your original string.

```cpp
#include "ResponseBuilder.hpp"
#include <sstream>

HttpResponse ResponseBuilder::build(const HttpRequest& request)
{
    (void)request; // unused for now

    HttpResponse response;

    std::string body = "Hello pink t-shirts team";

    response.setStatus(200, "OK");

    std::ostringstream len;
    len << body.size();

    response.setHeader("Content-Length", len.str());
    response.setHeader("Connection", "close");
    response.setHeader("Content-Type", "text/plain");

    response.setBody(body);

    return response;
}
```

# 5️⃣ How `ConnectionHandler` uses it

Inside your `handleRead()` when request becomes complete:

```cpp
ResponseBuilder builder;

response_ = builder.build(request_.getRequest());

write_buffer_ = response_.serialize();

write_offset_ = 0;

reactor_.updateHandler(this, EPOLLOUT | EPOLLET);
```

# 7️⃣ Why `serialize()` exists

Because HTTP is a **text protocol**.

Your internal structure:

```
status_code_ = 200
headers_ = map
body_ = string
```

must become a **single TCP stream**:

```
HTTP text format
```

That transformation is `serialize()`.

---

# 8️⃣ Why this design scales well

Later you can extend `ResponseBuilder`:

Example:

```
GET /           → index.html
GET /images     → file server
POST /upload    → upload handler
GET /cgi-bin    → CGI execution
```

Example future logic:

```cpp
if (request.getMethod() == "GET")
    return buildGet(request);

if (request.getMethod() == "POST")
    return buildPost(request);
```

---

# 9️⃣ Optional improvement (good practice)

You can automatically set `Content-Length` in `serialize()`:

Instead of requiring builder to set it.

Example:

```cpp
if (headers_.count("Content-Length") == 0)
{
    response << "Content-Length: " << body_.size() << "\r\n";
}
```

This prevents mistakes.

# 🔟 Resulting architecture (clean)

Your server now has a **very good separation**:

```
ConnectionHandler
    │
    ├── RequestParser
    │
    ├── ResponseBuilder
    │
    └── HttpResponse
            └── serialize()
```

This is **very close to real HTTP server architecture**.
