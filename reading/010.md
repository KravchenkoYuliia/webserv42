**Regarding `bad_alloc` from `push_back`:** Yes, `std::vector::push_back` automatically throws `std::bad_alloc` if memory allocation fails — you don't need to do anything special. Your current fix (push first, then `epoll_ctl`, then `pop_back` on failure) is correct.

**Issues I still see in your current code:**

**1. Event loop order bug — error check comes AFTER read/write**

```cpp
if (ready_events_list[i].events & EPOLLIN)
    handler->handleRead();   // handler may be deleted here!
if (ready_events_list[i].events & EPOLLOUT)
    handler->handleWrite();  // use-after-free if deleted above
if (ready_events_list[i].events & (EPOLLERR | EPOLLHUP))
{
    handler->handleError();
    continue; // too late, damage already done
}
```

The `continue` only skips nothing useful — error should be checked **first**.

---

**The "dead flag" pattern — full implementation:**

The idea: instead of deleting handlers mid-loop (which causes use-after-free), mark them as "pending deletion" and reap them after the loop completes.

```cpp
// IEventHandler.hpp
class IEventHandler {
public:
    IEventHandler() : dead_(false) {}
    virtual ~IEventHandler() {}

    virtual void handleRead()  = 0;
    virtual void handleWrite() = 0;
    virtual void handleError() = 0;
    virtual int  getFd() const = 0;

    bool isDead() const { return dead_; }
    void markDead()     { dead_ = true; }

private:
    bool dead_;
};
```

```cpp
// Reactor - deleteHandler becomes a two-step process
void Reactor::deleteHandler(int fd)
{
    // Step 1: remove from epoll immediately (fd may close after this)
    epoll_ctl(epoll_fd_, EPOLL_CTL_DEL, fd, NULL);

    // Step 2: mark dead instead of deleting — safe during event loop
    for (std::vector<IEventHandler*>::iterator it = handlers_.begin();
         it != handlers_.end(); ++it)
    {
        if ((*it)->getFd() == fd)
        {
            (*it)->markDead();
            break;
        }
    }
}

// New: call this after each epoll_wait iteration
void Reactor::reapDeadHandlers()
{
    for (std::vector<IEventHandler*>::iterator it = handlers_.begin();
         it != handlers_.end(); )
    {
        if ((*it)->isDead())
        {
            delete *it;
            it = handlers_.erase(it); // erase returns next valid iterator
        }
        else
            ++it;
    }
}
```

```cpp
void Reactor::run()
{
    const int kMaxReadyEventsBatchSize = 1024;
    struct epoll_event ready_events_list[kMaxReadyEventsBatchSize];

    while (true)
    {
        int num_fds_ready = epoll_wait(epoll_fd_, ready_events_list,
                                       kMaxReadyEventsBatchSize, -1);
        if (num_fds_ready == -1)
        {
            if (errno == EINTR)
                continue;
            throw std::runtime_error("epoll_wait failed");
        }

        for (int i = 0; i < num_fds_ready; ++i)
        {
            IEventHandler *handler =
                static_cast<IEventHandler *>(ready_events_list[i].data.ptr);

            // Skip if already marked dead by a previous handler this iteration
            if (handler->isDead())
                continue;

            uint32_t events = ready_events_list[i].events;

            // Error first — skip read/write if broken
            if (events & (EPOLLERR | EPOLLHUP))
            {
                handler->handleError(); // handleError calls deleteHandler internally
                continue;
            }
            if (events & EPOLLIN)
                handler->handleRead();
            if ((events & EPOLLOUT) && !handler->isDead()) // guard after read
                handler->handleWrite();
        }

        reapDeadHandlers(); // safe deletion AFTER the loop
    }
}
```

**Other useful flags to consider:**

| Flag | Purpose |
|---|---|
| `dead_` | Pending deletion, skip in event loop |
| `writable_` | Track if handler is registered for `EPOLLOUT` to avoid redundant `updateHandler` calls |
| `type_` (enum) | `ACCEPTOR` vs `CONNECTION` — avoids `dynamic_cast` when you need to distinguish handler types |

The `writable_` flag is particularly useful — you only want `EPOLLOUT` registered when you actually have data to send, otherwise epoll will busy-loop:

```cpp
void ConnectionHandler::enableWrite()
{
    if (!writable_) {
        writable_ = true;
        reactor_.updateHandler(this, EPOLLIN | EPOLLOUT);
    }
}

void ConnectionHandler::disableWrite()
{
    if (writable_) {
        writable_ = false;
        reactor_.updateHandler(this, EPOLLIN);
    }
}
```

In a webserv project you typically have exactly 3:

| Handler | Role | Flags needed |
|---|---|---|

## Full integration

**IEventHandler.hpp**
```cpp
#ifndef IEVENTHANDLER_HPP
#define IEVENTHANDLER_HPP

#include <stdint.h>

class IEventHandler
{
public:
    enum Type { ACCEPTOR, CONNECTION, CGI };

    IEventHandler(Type type)
        : dead_(false)
        , writable_(false)
        , type_(type)
    {}

    virtual ~IEventHandler() {}

    virtual void handleRead()  = 0;
    virtual void handleWrite() = 0;
    virtual void handleError() = 0;
    virtual int  getFd() const = 0;

    // dead flag — marks handler for deletion after event loop iteration
    bool isDead()   const { return dead_; }
    void markDead()       { dead_ = true; }

    // writable flag — tracks EPOLLOUT registration to avoid redundant epoll_ctl calls
    bool isWritable()      const { return writable_; }
    void setWritable(bool v)     { writable_ = v; }

    // type flag — avoids dynamic_cast when distinguishing handler kinds
    Type getType() const { return type_; }

private:
    bool dead_;
    bool writable_;
    Type type_;
};

#endif
```

**Reactor.cpp**
```cpp
#include "Reactor.hpp"
#include <iostream>

Reactor::Reactor(void)
{
    epoll_fd_ = epoll_create(1);
    if (epoll_fd_ == -1)
        throw std::runtime_error("epoll_create failed");
}

Reactor::~Reactor(void)
{
    for (std::vector<IEventHandler*>::iterator it = handlers_.begin();
         it != handlers_.end(); ++it)
        delete *it;
    handlers_.clear();
    if (epoll_fd_ != -1)
        close(epoll_fd_);
}

void Reactor::addHandler(IEventHandler *handler, uint32_t events)
{
    if (handler == NULL)
        throw std::invalid_argument("handler is NULL");

    handlers_.push_back(handler); // push first so ~Reactor() can clean on failure

    int fd = handler->getFd();
    struct epoll_event event;
    event.events   = events;
    event.data.ptr = handler;

    if (epoll_ctl(epoll_fd_, EPOLL_CTL_ADD, fd, &event) == -1)
    {
        handlers_.pop_back(); // undo — caller still owns the pointer
        throw std::runtime_error("epoll_ctl ADD failed");
    }
}

void Reactor::updateHandler(IEventHandler *handler, uint32_t events)
{
    int fd = handler->getFd();
    struct epoll_event event;
    event.events   = events;
    event.data.ptr = handler;

    if (epoll_ctl(epoll_fd_, EPOLL_CTL_MOD, fd, &event) == -1)
        throw std::runtime_error("epoll_ctl MOD failed");
}

// Marks dead + removes from epoll — does NOT delete (reapDeadHandlers does that)
void Reactor::deleteHandler(int fd)
{
    epoll_ctl(epoll_fd_, EPOLL_CTL_DEL, fd, NULL); // ignore error — fd may be closed

    for (std::vector<IEventHandler*>::iterator it = handlers_.begin();
         it != handlers_.end(); ++it)
    {
        if ((*it)->getFd() == fd)
        {
            (*it)->markDead();
            break;
        }
    }
}

// Convenience: adds EPOLLOUT to a handler only if not already registered
void Reactor::enableWrite(IEventHandler *handler)
{
    if (!handler->isWritable())
    {
        handler->setWritable(true);
        updateHandler(handler, EPOLLIN | EPOLLOUT);
    }
}

// Convenience: removes EPOLLOUT when send buffer is drained
void Reactor::disableWrite(IEventHandler *handler)
{
    if (handler->isWritable())
    {
        handler->setWritable(false);
        updateHandler(handler, EPOLLIN);
    }
}

void Reactor::reapDeadHandlers()
{
    for (std::vector<IEventHandler*>::iterator it = handlers_.begin();
         it != handlers_.end(); )
    {
        if ((*it)->isDead())
        {
            delete *it;
            it = handlers_.erase(it);
        }
        else
            ++it;
    }
}

void Reactor::run()
{
    const int kMaxEvents = 1024;
    struct epoll_event ready[kMaxEvents];

    while (true)
    {
        int n = epoll_wait(epoll_fd_, ready, kMaxEvents, -1);
        if (n == -1)
        {
            if (errno == EINTR)
                continue;
            throw std::runtime_error("epoll_wait failed");
        }

        for (int i = 0; i < n; ++i)
        {
            IEventHandler *handler =
                static_cast<IEventHandler*>(ready[i].data.ptr);

            if (handler->isDead()) // already condemned this iteration
                continue;

            uint32_t ev = ready[i].events;

            if (ev & (EPOLLERR | EPOLLHUP))
            {
                handler->handleError(); // internally calls deleteHandler
                continue;              // skip read/write
            }
            if (ev & EPOLLIN)
                handler->handleRead();
            if ((ev & EPOLLOUT) && !handler->isDead()) // guard: read may have killed it
                handler->handleWrite();
        }

        reapDeadHandlers(); // safe point — no iterators active
    }
}
```

```cpp
ConnectionAcceptor::ConnectionAcceptor(Socket *socket, Reactor &reactor)
    : IEventHandler(IEventHandler::ACCEPTOR) // sets type_
    , socket_(socket)
    , reactor_(reactor)
{}

void ConnectionAcceptor::handleRead()
{
    int client_fd = socket_->accept();
    if (client_fd == -1)
        return; // EAGAIN — no connection waiting

    Utils::setNonBlocking(client_fd);

    ConnectionHandler *handler = new ConnectionHandler(client_fd, reactor_);
    try {
        reactor_.addHandler(handler, EPOLLIN);
    }
    catch (const std::exception &e) {
        delete handler; // addHandler failed, we still own it
        throw;
    }
}

void ConnectionAcceptor::handleError()
{
    // Acceptor errors are fatal — shut down
    reactor_.deleteHandler(socket_->getFd()); // marks dead_, removed from epoll
}

void ConnectionAcceptor::handleWrite() {} // acceptor never writes
```

```cpp
    : IEventHandler(IEventHandler::CONNECTION) // sets type_
    , fd_(fd)
    , reactor_(reactor)
{}

ConnectionHandler::~ConnectionHandler()
{
    if (fd_ != -1)
        close(fd_);
}

void ConnectionHandler::handleRead()
{
    char buf[4096];
    ssize_t n = recv(fd_, buf, sizeof(buf), 0);

    if (n == 0 || (n < 0 && errno != EAGAIN))
    {
        reactor_.deleteHandler(fd_); // marks dead_, safe mid-loop
        return;
    }
    if (n < 0) return; // EAGAIN — nothing to read yet

    // TODO: feed buf into HTTP parser
    // Once response is ready:
    reactor_.enableWrite(this); // registers EPOLLOUT only when needed
}

void ConnectionHandler::handleWrite()
{
    // TODO: send pending response data
    ssize_t n = send(fd_, response_buf_.data(), response_buf_.size(), 0);

    if (n < 0 && errno != EAGAIN)
    {
        reactor_.deleteHandler(fd_);
        return;
    }
    // If all data sent — stop watching EPOLLOUT to avoid busy-loop
    if (response_buf_.empty())
        reactor_.disableWrite(this); // removes EPOLLOUT, keeps EPOLLIN
}

void ConnectionHandler::handleError()
{
    reactor_.deleteHandler(fd_);
}
```

## Flag summary for your project

| Flag | Lives in | Used by | Why |
|---|---|---|---|
| `dead_` | `IEventHandler` | All 3 handlers | Safe deletion mid-loop |
| `writable_` | `IEventHandler` | `ConnectionHandler`, `CGIHandler` | Avoid `EPOLLOUT` busy-loop |
| `type_` | `IEventHandler` | Reactor / debug | Avoid `dynamic_cast` |
| `cgi_done_` | `CGIHandler` only | `CGIHandler` | CGI pipe EOF before response flushed |

`ConnectionAcceptor` only ever reads (new connections), so it never needs `writable_`. The `type_` flag is optional but saves you from ugly casts when you need to differentiate handlers in the reactor or in logging.
