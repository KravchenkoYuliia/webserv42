# 🧠 1. Where to integrate it (IMPORTANT)

👉 Do **NOT** mix it into `parseBodyChunked()` or `parseBodyContentLength()`

Those should ONLY:

* reconstruct the full body

## ✅ Correct place in your flow

Right after body is complete:

# 🧩 2. Add a simple handler inside `RequestParser`

```cpp
void RequestParser::handleMultipartIfNeeded()
{
    const std::string& content_type = request_.getHeader(Http::Headers::CONTENT_TYPE);

    if (content_type.find("multipart/form-data") == std::string::npos)
        return;

    parseMultipartBody();
}
```

# 🧱 3. Minimal multipart parser (inside RequestParser)

## 🔹 Step 1: Extract boundary

```cpp
std::string RequestParser::extractBoundary(const std::string& content_type)
{
    size_t pos = content_type.find("boundary=");
    if (pos == std::string::npos)
        return "";

    return "--" + content_type.substr(pos + 9);
}
```

---

## 🔹 Step 2: Main parsing function

```cpp
void RequestParser::parseMultipartBody()
{
    const std::string& body = request_.getBody();
    std::string boundary = extractBoundary(request_.getHeader(Http::Headers::CONTENT_TYPE));

    if (boundary.empty())
        return;

    size_t pos = 0;

    while (true)
    {
        size_t start = body.find(boundary, pos);
        if (start == std::string::npos)
            break;

        start += boundary.length();

        // end marker
        if (body.substr(start, 2) == "--")
            break;

        // skip CRLF
        if (body.substr(start, 2) == "\r\n")
            start += 2;

        size_t end = body.find(boundary, start);
        if (end == std::string::npos)
            break;

        std::string part = body.substr(start, end - start);

        parseMultipartPart(part);

        pos = end;
    }
}
```

---

## 🔹 Step 3: Parse each part

```cpp
void RequestParser::parseMultipartPart(const std::string& part)
{
    size_t header_end = part.find("\r\n\r\n");
    if (header_end == std::string::npos)
        return;

    std::string headers = part.substr(0, header_end);
    std::string body = part.substr(header_end + 4);

    // remove trailing CRLF
    if (body.size() >= 2 && body.substr(body.size() - 2) == "\r\n")
        body.erase(body.size() - 2);

    if (headers.find("filename=") != std::string::npos)
    {
        std::string filename = extractFilename(headers);
        saveFile(filename, body);
    }
}
```

---

## 🔹 Step 4: Extract filename

```cpp
std::string RequestParser::extractFilename(const std::string& headers)
{
    size_t pos = headers.find("filename=\"");
    if (pos == std::string::npos)
        return "upload.bin";

    pos += 10;
    size_t end = headers.find("\"", pos);
    return headers.substr(pos, end - pos);
}
```

---

## 🔹 Step 5: Save file

```cpp
void RequestParser::saveFile(const std::string& filename, const std::string& data)
{
    std::ofstream out(filename.c_str());
    out << data;
    out.close();
}
```

# ⚠️ 4. Important design warnings

Even if you keep it simple:

---

## ❌ Don’t do this

* Parse multipart during chunk parsing
* Parse multipart before full body is ready

👉 That will break EVERYTHING

---

## ⚠️ This is acceptable (for 42)

* parsing inside `RequestParser` ✔️
* writing file directly ✔️
* ignoring non-file fields ✔️

---

## ❗ But keep logic separated mentally:

```text
HTTP parsing → DONE
multipart parsing → AFTER
```

Even if in same class.

# 🧠 Final takeaway

You **can** keep everything in `RequestParser`, as long as:

* multipart parsing happens **after COMPLETE**
* you **don’t mix it with HTTP parsing**
* you treat it as a **second step**
