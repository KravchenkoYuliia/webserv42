## 1. Where to implement epoll()

Since epoll is at the **I/O multiplexing** level, it doesn’t belong in `Socket` (which is just a wrapper around a file descriptor), nor in HTTP parsing or routing. It’s part of the **core event loop**.

Looking at your structure:

```
include/core/
├── ServerManager.hpp
├── EventLoop.hpp
├── Connection.hpp
├── ConnectionManager.hpp
├── Socket.hpp
```

The best placement:

* **EventLoop.hpp / EventLoop.cpp** – This class should own epoll and handle the main loop.
  Responsibilities:

  * Initialize epoll (`epoll_create`)
  * Register file descriptors (`epoll_ctl`)
  * Wait for events (`epoll_wait`)
  * Dispatch read/write events to `Connection` objects

* `ServerManager` can manage multiple servers/listening sockets, but the **reactor pattern** itself (polling & dispatching) is handled in `EventLoop`.

So, a high-level diagram:

```
ServerManager -> holds multiple listening Sockets
EventLoop -> uses epoll to wait for events
ConnectionManager -> manages active connections
Connection -> represents one client connection
```

### Example: EventLoop with epoll (C++98 style)

```cpp
#include <sys/epoll.h>
#include <unistd.h>
#include <vector>
#include "Connection.hpp"
#include "ConnectionManager.hpp"

class EventLoop {
private:
    int epoll_fd;
    std::vector<Connection*> activeConnections;
    ConnectionManager &connManager;

public:
    EventLoop(ConnectionManager &manager)
        : connManager(manager)
    {
        epoll_fd = epoll_create(1024);
        if (epoll_fd == -1) {
            perror("epoll_create");
            exit(1);
        }
    }

    ~EventLoop() {
        close(epoll_fd);
    }

    void addConnection(Connection *conn) {
        struct epoll_event ev;
        ev.events = EPOLLIN | EPOLLET; // read events, edge-triggered
        ev.data.ptr = conn; // store the connection object
        if (epoll_ctl(epoll_fd, EPOLL_CTL_ADD, conn->getFd(), &ev) == -1) {
            perror("epoll_ctl ADD");
        }
        connManager.addConnection(conn);
    }

    void run() {
        const int MAX_EVENTS = 1024;
        struct epoll_event events[MAX_EVENTS];

        while (true) {
            int n = epoll_wait(epoll_fd, events, MAX_EVENTS, -1);
            if (n == -1) {
                perror("epoll_wait");
                continue;
            }

            for (int i = 0; i < n; ++i) {
                Connection *conn = static_cast<Connection*>(events[i].data.ptr);

                if (events[i].events & EPOLLIN) {
                    conn->handleRead();
                }
                if (events[i].events & EPOLLOUT) {
                    conn->handleWrite();
                }
            }
        }
    }
};
```

Key points:

* `EventLoop` owns **epoll_fd**.
* `ConnectionManager` just keeps track of connections.
* `Connection` implements **read/write handlers** (wrapping `recv`/`send`).

## 2. How to use Connection and ConnectionManager

* **Connection**: Represents a client socket. Holds a `Socket`, read/write buffers, and request parsing state.
* **ConnectionManager**: Keeps a collection of `Connection` objects, helps with cleanup, lookup, and iteration.

### Example: Connection

```cpp
class Connection {
private:
    Socket sock;
    std::string readBuffer;
    std::string writeBuffer;

public:
    Connection(int fd) : sock(fd) {}

    int getFd() const { return sock.getFd(); }

    void handleRead() {
        char buffer[4096];
        int n = sock.recv(buffer, sizeof(buffer));
        if (n > 0) {
            readBuffer.append(buffer, n);
            // You could parse HTTP requests here
        } else if (n == 0) {
            // Client disconnected
            sock.close();
        }
    }

    void handleWrite() {
        if (!writeBuffer.empty()) {
            int n = sock.send(writeBuffer.c_str(), writeBuffer.size());
            if (n > 0) {
                writeBuffer.erase(0, n);
            }
        }
    }

    void queueResponse(const std::string &response) {
        writeBuffer += response;
    }
};
```

### Example: ConnectionManager

```cpp
#include <map>

class ConnectionManager {
private:
    std::map<int, Connection*> connections;

public:
    void addConnection(Connection *conn) {
        connections[conn->getFd()] = conn;
    }

    void removeConnection(int fd) {
        std::map<int, Connection*>::iterator it = connections.find(fd);
        if (it != connections.end()) {
            delete it->second;
            connections.erase(it);
        }
    }

    Connection* getConnection(int fd) {
        std::map<int, Connection*>::iterator it = connections.find(fd);
        return (it != connections.end()) ? it->second : NULL;
    }
};
```

## ✅ Recommended Flow

1. `ServerManager` creates listening sockets.
2. `EventLoop` registers them with **epoll**.
3. On new connection:

   * Accept the client
   * Create a `Connection`
   * Add it to `ConnectionManager`
   * Register it with epoll
4. On `EPOLLIN` or `EPOLLOUT` events, call `Connection::handleRead()` or `handleWrite()`.
