Excellent questions 👌 — this is exactly where most webserv designs become messy if not clarified early.

Let’s go one by one and make it concrete.

---

# 1️⃣ EventLoop::run(ConnectionManager&)

You asked:

> what is the purpose of run(connectionManager)?

Good catch — the name alone doesn’t explain the responsibility.

### 🔎 What `EventLoop` should NOT do

* It should NOT know about HTTP
* It should NOT parse requests
* It should NOT build responses

Its only job:

> Wait for events and dispatch them to the right owner.

So `run()` is basically:

```cpp
void EventLoop::run(ConnectionManager& connMgr)
{
    while (true)
    {
        int n = epoll_wait(_epollFd, _events.data(), MAX_EVENTS, -1);

        for (int i = 0; i < n; i++)
        {
            int fd = _events[i].data.fd;

            if (connMgr.isListeningSocket(fd))
                connMgr.handleNewConnection(fd);
            else
                connMgr.handleClientEvent(fd, _events[i].events);
        }
    }
}
```

### 🎯 Why pass `ConnectionManager&`?

Because:

* EventLoop detects events
* ConnectionManager knows what to do with them

This keeps responsibilities clean:

| Class             | Responsibility  |
| ----------------- | --------------- |
| EventLoop         | Detect events   |
| ConnectionManager | React to events |

You could rename it:

```cpp
void run(ConnectionManager& dispatcher);
```

If that feels clearer.

# 2️⃣ Socket::getFd() — which fd?

Excellent question.

There are TWO types of sockets in your server:

1. 🟢 Listening socket (server socket)
2. 🔵 Client socket (accepted connection)

Your `Socket` class should represent only the listening socket.

So:

```cpp
class Socket
{
private:
    int _fd;  // LISTENING socket fd

public:
    int getFd() const;
};
```

This fd refers to:

```
socket()
bind()
listen()
```

NOT the client.

Client fds are created by:

```cpp
int clientFd = accept(listenFd, ...);
```

Those should be stored inside `Connection` objects — not inside `Socket`.

# 3️⃣ HttpRequest + RequestParser interaction

Let’s clarify this with a real example.

You receive partial data from `recv()`:

```cpp
"GET /index.html HTTP/1.1\r\nHost: localhost\r\n\r\n"
```

## Inside Connection::handleRead()

```cpp
void Connection::handleRead()
{
    char buffer[4096];
    int bytes = recv(_fd, buffer, sizeof(buffer), 0);

    _readBuffer.append(buffer, bytes);

    if (_parser.parse(_readBuffer, _request))
    {
        _requestComplete = true;
    }
}
```

---

## RequestParser

```cpp
bool RequestParser::parse(std::string& buffer, HttpRequest& request)
{
    if (_state == REQUEST_LINE)
    {
        if (!containsCRLF(buffer))
            return false;

        parseRequestLine(buffer, request);
        _state = HEADERS;
    }

    if (_state == HEADERS)
    {
        if (!containsDoubleCRLF(buffer))
            return false;

        parseHeaders(buffer, request);
        _state = COMPLETE;
        return true;
    }

    return false;
}
```

### Important idea:

* `RequestParser` extracts data from `_readBuffer`
* It fills `HttpRequest`
* It updates internal state
* It returns true only when request is complete

---

## So interaction is:

| Object        | Role                                 |
| ------------- | ------------------------------------ |
| Connection    | Owns buffer + parser + request       |
| RequestParser | Converts buffer → structured request |
| HttpRequest   | Stores parsed data                   |

# 4️⃣ Routing — Let’s go deeper

Routing has 2 levels:

1️⃣ Select correct `ServerConfig`
2️⃣ Select correct `LocationConfig`

---

## Server matching

Based on:

* Port
* Host header

Example:

```nginx
server {
    listen 8080;
    server_name example.com;
}
```

When request comes:

```
Host: example.com
```

Router logic:

```cpp
const ServerConfig& Router::matchServer(
    const HttpRequest& request,
    int localPort)
{
    for (size_t i = 0; i < _servers.size(); i++)
    {
        if (_servers[i].port == localPort &&
            _servers[i].serverName == request.headers["Host"])
            return _servers[i];
    }

    return _defaultServer;
}
```

---

## Location matching (Longest prefix rule)

Example config:

```nginx
location /images
location /images/icons
location /
```

If URI is:

```
/images/icons/logo.png
```

You must choose:

```
/images/icons
```

### Implementation:

```cpp
const LocationConfig& Router::matchLocation(
    const ServerConfig& server,
    const std::string& uri)
{
    size_t bestMatchLen = 0;
    const LocationConfig* best = NULL;

    for (std::map<std::string, LocationConfig>::const_iterator it =
         server.locations.begin(); it != server.locations.end(); ++it)
    {
        const std::string& path = it->first;

        if (uri.compare(0, path.size(), path) == 0)
        {
            if (path.size() > bestMatchLen)
            {
                bestMatchLen = path.size();
                best = &it->second;
            }
        }
    }

    return *best;
}
```

# 5️⃣ Connection class — Full Design

This is the most important class in the whole project.

## It represents ONE client.

---

## It should hold:

```cpp
class Connection
{
private:
    int _fd;
    std::string _readBuffer;
    std::string _writeBuffer;

    RequestParser _parser;
    HttpRequest _request;
    HttpResponse _response;

    bool _requestComplete;
    bool _responseReady;
    bool _keepAlive;

public:
    Connection(int fd);

    void handleRead();
    void handleWrite();

    bool isRequestComplete() const;
    bool isResponseSent() const;

    int getFd() const;
};
```

---

## handleRead()

* recv
* append buffer
* parse
* if complete → build response

---

## handleWrite()

* send from `_writeBuffer`
* remove sent bytes
* if empty:

  * close OR
  * reset state (if keep-alive)

# 6️⃣ So what should ConnectionManager look like?

Yes — if you add `Connection`, you MUST update `ConnectionManager`.

---

## ConnectionManager should hold:

```cpp
class ConnectionManager
{
private:
    std::map<int, Connection*> _connections;
    std::set<int> _listeningSockets;

    EventLoop* _eventLoop;

public:
    void addListeningSocket(int fd);
    bool isListeningSocket(int fd) const;

    void handleNewConnection(int listenFd);
    void handleClientEvent(int clientFd, uint32_t events);

    void removeConnection(int fd);
};
```

---

## handleNewConnection()

```cpp
void ConnectionManager::handleNewConnection(int listenFd)
{
    int clientFd = accept(listenFd, ...);

    setNonBlocking(clientFd);

    Connection* conn = new Connection(clientFd);
    _connections[clientFd] = conn;

    _eventLoop->addFd(clientFd, EPOLLIN);
}
```

---

## handleClientEvent()

```cpp
void ConnectionManager::handleClientEvent(int fd, uint32_t events)
{
    Connection* conn = _connections[fd];

    if (events & EPOLLIN)
        conn->handleRead();

    if (conn->isRequestComplete())
    {
        buildResponse(conn);
        _eventLoop->modifyFd(fd, EPOLLOUT);
    }

    if (events & EPOLLOUT)
        conn->handleWrite();

    if (conn->isResponseSent())
        removeConnection(fd);
}
```
