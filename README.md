# Webserv - our own HTTP server in C++98
HTTP is one of the most widely used protocols on the internet.     
The Hypertext Transfer Protocol ( HTTP ) is an application protocol for distributed,
collaborative, hypermedia information systems.  

## Instruction
- `make` to compile 
- `./webserv config/default.conf` to start a server ( config file is just an example. You can modify it )
- connect a client via browser/telnet/curl

## Features
### Request types
`GET` - get smth from server  
`POST` - add smth to server  
`DELETE` - delete smth inside server

### Request via telnet/curl
example:
1. GET default page from server
```
GET / HTTP/1.0 


```
2. Add plain text to server. Will be added to the `root` inside location block `upload`

```
POST /upload/ HTTP/1.0
Content-Type: text/plain
Content-Length: 5

12345
```

3. Delete the file from the `root` inside the location block `upload` if file exists. Otherwise error page is return

```
DELETE /upload/text_uploaded_to_server.txt HTTP/1.0


```

### Host
There are multiple server blocks in config/default.conf with different configuration.  
Server reads the request and choose a corresponding server block configuration accordingly to `port`.  
If there several server blocks with the same port - choose accordingly 
`host: host_name` in request and server_name directive in config.  
If no `host` in request or there are multiple server blocks in config with the same port AND server_name - choose server with directive `default_server`.   
If no `default_server` directive - choose first server with corresponding `port`.

### Listing

If uri is a directory - server returns default page of this directory ( directive `index` in config ).  
If `index` doesn't exists - show a list of all files inside the directory ( inside the `root` of this location ). 
Listing is only possible if directive `autoindex` in config is `on`.  
![config_public]( /docs/assets/config_public.png )
![list_public]( /docs/assets/list_public.png )

### Error page

If any error occurs server is looking for directive `error_page` with corresponding error code inside location block to return error page to the client. 
If there's not - inside server block.  

If there is not error page provided - the default error page is generated by server.
